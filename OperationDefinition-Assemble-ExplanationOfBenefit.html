OperationDefinition: Assemble ExplanationOfBenefit

AssembleExplanationOfBenefit

OPERATION: AssembleExplanationOfBenefit

The official URL for this operation definition is: 

http://hl7.org/fhir/us/carin-bb/OperationDefinition/assemble-eob

This operation is used to return ExplanationOfBenefit resources related to a patient along with all the reference resources.

The operation takes the optional input parameters:

  • patient id
  • start date
  • end date
  • document type

and returns a Bundle of type "searchset" containing CARIN BB ExplanationOfBenefit and reference resources for the patient. The principle intended use for this operation is to provide a patient with access to their available ExplanationOfBenefit information.

URL: [base]/ExplanationOfBenefit/$assemble-eob

Parameters

UseNameCardinalityTypeBindingDocumentation
INpatient1..1id

The id of the patient resource located on the server on which this operation is executed. If there is no match, an empty Bundle is returned

IN_since0..1instant

ExplanationOfBenefit resources updated after this period will be included in the response. The intent of this parameter is to allow a client to request only records that have changed since the last request.

INtype0..1CodeableConcepthttp://hl7.org/fhir/ValueSet/claim-type (Required)

The type of the ExplanationOfBenefit, e.g. instituional, professional, pharmacy, etc.

INservice-date0..1date

Date of the service for the ExplanationOfBenefit.

INidentifier0..1Identifier

Payer Claim Number / Business Identifier of the ExplanationOfBenefit resource.

IN_id0..1id

Logical id for the ExplanationOfBenefit resource.

OUTreturn1..1Bundle

The bundle type is "searchset"containing CARIN BB ExplanationOfBenefit and reference resources. The bundle total will reflect the number of ExplanationOfBenefit instances returned.

  • The server is responsible for determining the reference resources to return as included resources rather than the client specifying which ones. This frees the client from needing to determine what it could or should ask for. The server SHALL return all the reference resources within the return bundle along with the ExplanationOfBenefit resources. The server's CapabilityStatement should document this behavior.
